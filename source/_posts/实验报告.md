#### MPI的派生数据类型

在分布式内存中，通信比本地计算开销大得多，因此减少发送的消息数量能够提高程序的性能。常用的整合多条消息的数据的三种手段为：1. 通信函数中的count、2. 派生数据类型、3. MPI_Pack/Unpack函数

在矩阵乘法的例子中，my_rank==0将matrix_A分发给所有的进程，每个进程则获得line行的数据。将这line行的数据（其实是一个line*size的矩阵）作为整体形成一种派生数据类型。

~~~c++
//定义MPI的数据类型——每个block:line×size的矩阵
MPI_Datatype aBlock;

vector<int> blocklengths(line,size); 			// 每个部分的数据个数  
vector<MPI_Datatype> types (line, MPI_INT); 	// 每个部分的数据类型  
vector<MPI_Aint> displs(line); 					// 每个部分的位移 
    for(int i=0;i<line;i++){
        displs[i]=i*size*sizeof(int);
    }

MPI_Type_create_struct(line, blocklengths.data(), displs.data(), types.data(), &aBlock); MPI_Type_commit(&aBlock);  
~~~



在0号进程进行分发：

~~~c++
//分发
for(int i=1;i<comm_sz;i++){
    //发送
    MPI_Send(matrix_A.data()+(i-1)*line*size, 1, aBlock, i, 1, MPI_COMM_WORLD);
}
~~~



其他进程接收：

~~~c++
// 接收
MPI_Recv(local_A.data(), 1, aBlock, 0, 1, MPI_COMM_WORLD, MPI_STATUS_IGNORE);
~~~

#### 性能及扩展性分析：

1. **Lab1的实验结果和集合通信的实验结果对比**，主要体现在集合通信和点对点通信上面。由于不再是只有主进程串行地发送和接收数据，使得通信成本降低，使得总体项目的运行时间变快。并且可以看出，随着矩阵规模的变大，加速越加明显。
2. **Lab1的实验结果和MPI派生类型的实验结果对比**，由于两者其实都是基于整合多条消息的数据的手段：count和派生数据类型，从结果上来看其实两者并无太大差异。貌似随着矩阵规模变大，后者比前者的表现会更好。



## 四、实验感想

实验过程中遇到的困难：

1. MPI的接口很多，参数也很多，准确理解和应用显得十分重要。实验过程中，对`MPI_Type_create_struct()`中的`array_of_displacements`的理解一直存在偏差。曾误以为为与每一项的真实地址、前一项的偏移差等，导致代码无法运行。更多的例子还有，`MPI_Send`和`MPI_Recv`中的参数`buf_size`，其代表着对应的存储对象的个数，所以当参数类型变成自定义的类型时，其也需要改变。
